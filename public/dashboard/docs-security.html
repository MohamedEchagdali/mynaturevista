Our Security Measures

MyNaturevista takes security seriously. We protect your account and your data with multiple layers of security:

  • All data is encrypted during transmission
  • Your passwords are securely stored (not visible to anyone, even us)
  • API keys are validated on every request
  • We use secure HTTPS connections everywhere
  • Regular security updates and monitoring

Your Password

Your password is your primary defense against unauthorized access to your account.

Creating a Strong Password

Password Best Practices
  • Use at least 8 characters
  • Mix uppercase and lowercase letters
  • Include numbers
  • Include special characters (!@#$%)
  • Avoid personal information (name, birthday, etc.)
  • Never use "password123" or similar common phrases

Changing Your Password

Change your password regularly for added security:

1

Go to Account Settings

Click "Account Settings" in the Account section

2

Click Change Password

Enter your current password and new password

3

Save

Click "Update Password". Your password will be updated immediately

Protecting Your API Keys

Your API keys control access to your widget. For security, each API key is restricted to specific domains, so even if a key is exposed, it can only be used on the domain it was created for.

How Domain-Restricted API Keys Work

Each API key is tied to one or more authorized domains. Requests coming from any other domain are automatically blocked.

API Key Safety Rules

NEVER:
  • Share your API key with others
  • Post it in public forums or social media
  • Store it in plain text or unencrypted emails
  • Commit it to public repositories
  • Attempt to use the same key across unrelated domains
BEST PRACTICES:
  • Create one API key per domain
  • Each key is automatically limited to its assigned domain
  • Revoke and regenerate keys if you suspect misuse
  • Delete API keys that are no longer needed

If You Think Your Key Is Compromised

1

Create a New Key

Go to the API Access section and generate a new API key for the affected domain.

2

Update Your Website

Replace the old API key with the new one on your website.

3

Revoke the Old Key

Delete or revoke the compromised API key to prevent further use.

Domain Security

Your API keys only work on the specific domains you register. This prevents someone else from stealing your key and using it on their website.

Always Use HTTPS

Make sure your website uses HTTPS (secure connection), not HTTP. Check that your URL starts with "https://" not "http://".

Registering Your Domains

When you create an API key, be specific about which domains will use it:

  • Enter exact domain: https://mywebsite.com
  • Or with www: https://www.mywebsite.com
  • Or all subdomains: https://*.mywebsite.com
Security Tip

Don't use overly broad domain patterns. Specify exact domains when possible to limit exposure if a key is compromised.

Your Data Protection

All your data is protected and encrypted. We don't sell or share your information.

What We Collect

  • Your name and email (for your account)
  • Widget usage statistics (how many times it's opened)
  • Custom location information (what you create)
  • Billing information (for processing payments)

What We DON'T Do

  • We never sell your data
  • We never share your data with third parties
  • We don't access your website's visitor data
  • We don't track users across websites
